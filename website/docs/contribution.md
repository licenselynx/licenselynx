# Contributing

LicenseLynx accepts contributions for data, code, and documentation.

The project is a monorepo with separate folders for the libraries, the data pipeline, and the website:

- `scripts/` contains the Python-based update, merge, and validation pipeline.
- `python/`, `java/`, and `typescript/` contain the libraries.
- `website/` contains the MkDocs documentation site.

The pages below cover the main contribution paths:

- [Adding Mappings](contributing/adding-mappings.md) for OSS license data.
- [Organization Mappings](contributing/organization-mappings.md) for organization-specific data under `data/orgs/`.
- [New Language Library](contributing/new-language-library.md) for adding support for another programming language.
- [Releases](contributing/releases.md) for versioning and release rules.

## Reporting Issues

If you detect a bug in the software or find an error in the data, create an issue.
For feature requests, create a new issue if one does not already exist.
